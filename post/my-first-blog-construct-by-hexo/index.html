My first Blog construct by hexo

2020-03-09 4 min read # Blog

​ 本着踩边所有坑的态度历经两天建成了自己的第一个博客，当然这是第一篇文章，可见此文章的地位非同一般。果不其然，其间遇见了各路小鬼在太岁头上动土。包括但不限于网络问题无法上传至根服务器，没有网站前端知识导致的修改代码问题.....遍历了所有办法几乎是十八般武艺全都用上...第一篇文章测试一下网站的标签功能是否正常，以及建立起一套标准的写文章到发文章的可靠路径，毕竟不是什么时间都有大把精力用在客制化主题上，博客还是主要以记录为目的的。

全局代码

hexo init        ####安装
hexo g           ###生成
hexo clean       ###清除缓存
hexo d           ###上传

更改git代理

git config --global http.proxy http://127.0.0.1:10809
git config --global https.proxy https://127.0.0.1:10809

取消代理

git config --global --unset http.proxy
git config --global --unset https.proxy

按装一个插件来同步：

npm install hexo-deployer-git --save

git yilia主题

git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

在blog/_config.yml中加入themes信息和github库信息

type: git
repo: https://github.com/heroamd/heroamd.github.io.git
branch: master

在blog\source加入CNAME文件以改变域名，这个路径里面的所有文件都将会生成并放置在网站的根目录下。

tips:所有代码前面不能加空格~~

生成SSH添加到GitHub

git config --global user.name "heroamd"
git config --global user.email "icenet11@163.com"

检查是否正确

git config user.email

配置ssh

ssh-keygen -t rsa -C "icenet11@163.com"

连接一下服务器测试是否成功

ssh -T git@github.com

如果提示

Hi heroamd! You've successfully authenticated, but GitHub does not provide shell access.

表示成功！！

cnpm install --save hexo-deployer-git

185.199.111.153

备份一下本电脑的代理端口排除的目录

localhost;127.*;10.*;172.16.*;172.17.*;172.18.*;172.19.*;172.20.*;172.21.*;172.22.*;172.23.*;172.24.*;172.25.*;172.26.*;172.27.*;172.28.*;172.29.*;172.30.*;172.31.*;192.168.*;windows10.microdone.cn

修改图片和头像

themes/yilia/source/下，可添加一个 assets 文件夹，里面存放图片资源即可

# 微信二维码图片
weixin:  /assets/wechat.png
# 头像图片
avatar:  /assets/head.jpg
# 网页图标
favicon:  /assets/head.jpg

所有文章模块的更改

1、请确保node版本大于6.2
2、在博客根目录（注意不是yilia根目录）执行以下命令：
npm i hexo-generator-json-content --save

3、在根目录_config.yml里添加配置：

  jsonContent:
    meta: false
    pages: false
    posts:
      title: true
      date: true
      path: true
      text: false
      raw: false
      content: false
      slug: false
      updated: false
      comments: false
      link: false
      permalink: false
      excerpt: false
      categories: false
      tags: true
