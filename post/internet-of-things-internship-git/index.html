Internet of Things internship: GIT

2021-07-08 2 min read # IoT # Tools

1、首先明确一个新的仓库管理平台需要先完成个人计算机SSH公钥的添加，也称为部署密钥，在生成SSH密钥之后（生成密钥），将.pub结尾的文件内容复制进目标平台即可完成认证

在云端添加SSH密钥（仓库》仓库设置）：

2、步骤一：初始化仓库

git init

3、步骤二：添加文件到上传，将目录下所有文件添加进上传记录

git add --all

4、步骤三：为新的推送添加评论，内容为first commit，一般内容为修改了什么东西或者处理了什么BUG

git commit -m "first commit"

5、步骤四：推送到远端master分支

第一次使用：

git remote add origin http://git.gaokeyun.cn/qixiuyuan2018080516/qixiuyuan2018080516.git

这条指令的意义是将origin这个名字背后的地址设置为http://git.gaokeyun.cn/qixiuyuan2018080516/qixiuyuan2018080516.git，也就是之后git push -u origin时此处的origin=仓库的地址

之后可以简写：

git push -u origin master

这句指令的意思是将本地的仓库推送到origin的master分支下面

然后就推送完毕了。

遇到的问题：

原因是上一步执行了 git add README.md，相当于除了README.md都不会上传到Git服务器，需要取消限制 执行

git add --all

原因是仓库中README.md文件不在本地代码目录中

解决办法使用git pull指令合并云端仓库和本地仓库

git pull --rebase origin master
