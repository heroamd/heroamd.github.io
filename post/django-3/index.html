1.1 网络相关原理

在应用层和传输层之间有一个虚拟层 》socket 套接字>相当于一个接口，在开发时只需要调用接口就可以。

C/S架构和B/S架构：customer to servers/ brossow to servers

S相当于远端服务器，用户端相当于B或者S，区别在于用户端是浏览器还是客户端。

远端服务器：

  1. socket服务端
  2. 绑定IP和端口
  3. 监听
  4. 等待连接
  5. 接受连接
  6. 接收数据
  7. 返回数据
  8. 断开连接

浏览器（socket客户端）

  1. socket客户端
  2. 连接百度上的socket服务端
  3. 发送数据
  4. 接收数据
  5. 断开连接

1.3 HTTP状态码

状态码的第一个数字代表当前响应的类型

  • 1xx——请求被服务器接受，继续处理
  • 2xx——请求成功被服务器接受并且接受
  • 3xx——重定向，需要后续操作才能处理
  • 4xx——请求错误，请求中含有错误无法被执行
  • 5xx——服务器处理请求发生错误
